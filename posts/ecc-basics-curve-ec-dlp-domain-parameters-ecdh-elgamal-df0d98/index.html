타원곡선암호 기초: 곡선 정의, EC-DLP, 도메인 파라미터, ECDH·EC-ElGamal

November 4, 2025

개요#

RSA와 ElGamal은 보안 강도를 유지하려면 시간이 갈수록 키 길이와 연산 복잡도가 커지는 경향이 있음. 계산 성능 향상과 공격 알고리즘 발전으로, 패딩 도입과 긴 키가 필수화되는 흐름이 생김. 이에 대한 대안으로 1985년에 제안된 타원곡선암호 ECC가 짧은 키 길이로 동등한 보안 수준과 빠른 연산을 제공함. ECC는 새로운 수학적 난제를 도입한 것이 아니라, 이산 로그 문제 DLP를 타원곡선 위에서 풀기가 더 어렵다는 점을 활용하는 구조임

이 글에서는 타원곡선의 정의와 군 구조, 유한체 위 곡선, EC-DLP, 도메인 파라미터, 그리고 ECDH·EC-ElGamal까지 핵심 흐름을 정리함

타원곡선과 표준형#

타원곡선은 3차 곡선을 적절히 변환한 바이어슈트라스 방정식 형태로 다루는 것이 일반적임. 체의 표수 characteristic p가 2와 3이 아닐 때, 곡선은 y^2 = x^3 + A x + B 형태로 단순화 가능. 이때 판별식 4A^3 + 27B^2 ≠ 0 조건을 만족해야 특이점이 없고 군 연산을 안정적으로 정의할 수 있음. 특이점이 존재하면 점 덧셈이 제대로 정의되지 않으며 안전한 암호 구조를 보장하기 어려움

표수가 2나 3인 체는 식과 연산식이 복잡해지고, 특정 공격 기법이 알려져 있어 일반적으로 배제하는 추세. 충분히 큰 홀수 표수의 체에서 표준형을 사용하면 구현 단순성과 알려진 안전성 가이드 모두를 충족하기 쉬움

군 연산과 성질#

타원곡선에는 무한원점 O를 항등원으로 도입. 점 덧셈은 기하적으로 정의하며 다음 성질을 만족하는 가환군 구조를 이룸

  • 닫힘성: P, Q가 곡선 위 점이면 P + Q도 곡선 위 점
  • 항등원: 임의의 P에 대해 P + O = P
  • 역원: 임의의 P에 대해 −P가 존재하고 P + (−P) = O
  • 교환·결합법칙: P + Q = Q + P, P + (Q + R) = (P + Q) + R

스칼라 곱 n·P는 P를 n번 더한 것으로 정의. 구현에서는 점 덧셈과 점 두배 연산을 조합해 이진법 스칼라 곱 등으로 효율화함

유한체 F_p 위의 타원곡선#

암호에서 일반적으로 유한체 F_p 위 곡선을 사용. 곡선 E(F_p)는 다음 집합으로 정의

  • E(F_p) = { (x, y) ∈ F_p^2 | y^2 ≡ x^3 + A x + B (mod p), 4A^3 + 27B^2 ≠ 0 (mod p) } ∪ { O }

E(F_p)는 위 점 덧셈에 대해 가환군을 이룸. 실수체 대비 부동소수점 이슈가 없고, 연산이 모듈러 정수 연산으로 닫혀 있어 실제 암호 구현에 적합함

EC-DLP#

타원곡선 이산 로그 문제 EC-DLP 정의

  • P, Q ∈ E(F_p)에서 Q = [d]P를 만족하는 정수 d를 찾는 문제

현대 알고리즘으로 알려진 최선은 지수 시간대에 가까운 성능으로, 충분히 큰 파라미터를 선택하면 해결이 사실상 불가능하다는 가정을 사용함

점 덧셈의 대수식#

P = (x1, y1), Q = (x2, y2) ∈ E(F_p), R = P + Q = (xR, yR). 모듈러 역원은 F_p에서 계산

  • P ≠ Q인 경우의 기울기 m
    • m = (y2 − y1) · (x2 − x1)^−1 mod p
  • P = Q인 경우의 기울기 m (점 두배)
    • m = (3 x1^2 + A) · (2 y1)^−1 mod p
  • 좌표 갱신
    • xR = m^2 − x1 − x2 mod p
    • yR = m (x1 − xR) − y1 mod p

특수 케이스로 x1 = x2이며 y1 = −y2인 경우 P + Q = O가 됨. 이 식들로 덧셈과 두배 연산을 구현하고, 스칼라 곱은 double-and-add 같은 알고리즘으로 구성함

점의 개수와 순환 부분군#

E(F_p)의 전체 점 개수는 곡선의 위수 N으로 표기. Schoof 계열 알고리즘으로 N을 구함. 실사용에서는 곡선 전체 군에서 위수 n의 순환 부분군을 잡고, 그 생성원 G를 도메인 파라미터로 사용

간단 예시로 y^2 = x^3 + 2x + 3 (mod 97)에서 P = (3, 6)이라 하자. P의 배수를 나열하면 다음과 같은 순환 부분군을 확인 가능

  • 0·P = O
  • 1·P = (3, 6)
  • 2·P = (80, 10)
  • 3·P = (80, 87)
  • 4·P = (3, 91)
  • 5·P = O

이 경우 P는 위수 5의 부분군 생성원. 라그랑주 정리에 의해 부분군의 위수 n은 전체 위수 N의 약수. 보조인자 cofactor는 h = N / n로 정의

도메인 파라미터#

ECC에서 연산 도메인을 지정하는 파라미터 집합

  • p: 유한체 F_p의 소수 모듈러스
  • A, B: 곡선 y^2 = x^3 + A x + B의 계수. 4A^3 + 27B^2 ≠ 0 (mod p)
  • G: 위수 n의 순환 부분군 생성점
  • n: ord(G). 충분히 큰 소수 선택 권장
  • h: 보조인자 h = N / n. N은 E(F_p)의 전체 위수

도메인 파라미터 구성 절차 요약

  1. Schoof 계열 알고리즘으로 곡선의 전체 위수 N 산출
  2. N의 약수 중 소수 n 선택. 목표 보안 강도에 부합해야 함
  3. h = N / n 계산
  4. 곡선 위 임의 점 P를 선택
  5. G = h · P 계산. G = O이면 4)로 돌아가 재시도
  6. G가 위수 n을 갖는지 검증. ord(G) = n 확인

실사용은 NIST 곡선군이나 secp256k1 등 검증된 표준 파라미터 채택 권장. 잘못된 도메인 파라미터는 구조적 취약점을 유발할 수 있음

ECDH 키 합의#

동일한 도메인 파라미터 {p, A, B, G, n, h}를 공유한다고 가정. ord(G) = n

  1. 참여자 각각 비밀 스칼라 선택: 1 ≤ α, β ≤ n − 1
  2. 공개키 계산 및 교환: A = αG, B = βG
  3. 공유 비밀 계산
    • 한쪽은 S = αB = α(βG) = (αβ)G
    • 상대는 S = βA = β(αG) = (αβ)G

S 자체 또는 S의 x좌표 등을 KDF에 넣어 세션 키 파생. 개인키 누설 없이 동일한 공유 비밀을 얻는 구조

EC-ElGamal 암호화#

동일 도메인 파라미터와 수신자 공개키 B = βG 공유 가정. 평문 M은 곡선 위 점으로 임베딩되었다고 가정. 실제 시스템에서는 보통 하이브리드 구성을 사용해 KDF로 대칭키를 만든 뒤 메시지를 대칭키로 암호화하는 방식이 일반적임

암호화 과정

  1. 송신자 무작위 k 선택. 1 ≤ k ≤ n − 1
  2. 암호문 C = (C1, C2) 구성
    • C1 = kG
    • C2 = M + kB

복호화 과정

  • 수신자는 β를 이용해 C2 − βC1 = M + k(βG) − β(kG) = M을 계산하여 평문 복원

ElGamal의 확률적 암호화 특성은 k의 무작위성에서 기인. k 재사용 금지와 고품질 난수원이 필수 요구 사항

마무리#

ECC는 점 군의 가환 구조와 EC-DLP의 난해성 위에서 짧은 키로 높은 보안 강도를 제공하는 공개키 암호 프리미티브를 구성함. 안전한 도메인 파라미터 선택, 올바른 점 연산 구현, 표준 곡선 사용이 핵심. 다음 단계로 해시와 디지털 서명을 다루며, 실전 프로토콜에서의 ECC 활용을 연결할 예정
