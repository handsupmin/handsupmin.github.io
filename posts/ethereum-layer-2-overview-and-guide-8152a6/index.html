이더리움 L2의 개념과 동작 원리, 선택 기준 정리

October 12, 2025

개념과 배경#

이더리움 같은 L1은 보안과 분산성 우선 설계로 처리량이 제한적임 초당 트랜잭션 수가 낮고 수수료가 높아 단건 송금이나 소액 결제에 비효율 발생 여러 트랜잭션을 묶어 처리하고 최소 정보만 L1에 게시해 확장성을 얻는 접근이 L2의 핵심 아이디어임 최근 EIP-4844로 L2 데이터 게시 비용이 큰 폭으로 하락해 L2의 경제성이 더 높아짐

계층 정리#

레이어 0 L0 블록체인 네트워킹·합의 인프라 기반, 체인 간 메시지 라우팅과 상호운용 지향 레이어 1 L1 자체 합의로 트랜잭션 최종성 확보, 보안의 근간 역할 레이어 2 L2 L1 위에서 실행되는 확장 계층, 트랜잭션을 모아 처리 후 압축 정보와 증명을 L1에 게시 레이어 3 L3 L2 위 애플리케이션 레이어, Dapp과 도메인 특화 기능 집약 참고 L2는 보안을 L1에 의존한다는 점이 핵심이며, 별도 합의로 운영되는 사이드체인은 보안 가정이 다름

L2의 동작 원리#

1 트랜잭션 수집 사용자가 L2로 전송, 시퀀서가 순서 지정 및 블록 생성 2 집계와 실행 L2 VM에서 상태 전이를 계산, 결과를 배치로 묶음 3 L1 게시 압축된 데이터와 메타데이터를 L1에 기록, 데이터 가용성 확보 4 검증 롤업 유형에 따라 보증 방식이 다름

  • 옵티미스틱 롤업 거래가 유효하다고 가정 후 도전 기간 동안 사기증명으로 반증 가능
  • ZK 롤업 각 배치에 대해 유효성 증명 제출, 짧은 최종성 제공 데이터 가용성 DA 모드가 비용과 보안을 좌우함
  • 온체인 DA L1에 데이터 게시, 보안 강하지만 비용 높음
  • 대안 DA 별도 DA 레이어 사용, 비용 절감 가능하나 보안 가정 추가

성능과 비용의 직관#

L2는 수천 건의 트랜잭션을 하나의 L1 게시로 압축 가능 L1 수수료 피크 타임에는 L2 게시도 비싸질 수 있으나, 단건당 평균 비용은 여전히 L1 대비 크게 낮음 ZK 롤업은 증명 생성 비용이 크지만 대규모 배치에서 단가가 낮아지는 경향

L2 선택 기준#

보안 가정 L1 보안 상속 여부, DA 위치, 검증자 집합의 탈중앙화 정도 최종성 요구 출금 지연 허용 시 옵티미스틱, 빠른 결제 필요 시 ZK 선호 EVM 호환성 기존 Solidity 자산 재사용 여부, 프리미티브 지원 범위 수수료와 TPS 목표 사용자 규모, 마이크로 결제 여부, 배치 크기와 지연 허용치 개발자 경험 툴체인, 디버깅, 인덱싱, 모니터링 생태계 거버넌스와 업그레이드 프로토콜 변경 절차, 보안 카운슬 권한, 긴급 중단 리스크

주의사항과 운영 팁#

브릿지 리스크 보안 중요 자산은 공식 브릿지 우선, 다중 서명 또는 서드파티 브릿지 신뢰 가정 점검 출금 대기 옵티미스틱 롤업은 L1 출금 지연 고려, 유동성 브릿지와의 수수료·슬리피지 트레이드오프 설계 시퀀서 단일점 시퀀서 장애 대비 폴백과 재시도 로직 준비, 지연 허용치 명시 가스 급등 L1 가스 스파이크 시 L2 게시 비용 상승, 배치 크기 동적 조정과 버퍼 예산 확보 데이터 가용성 대안 DA 사용 시 체인 정지나 재구성 상황 매뉴얼 준비 모니터링 L1·L2 체인 상태, 브릿지 큐, 도전 창, 증명 제출 지연을 별도 지표로 관측

간단 예시 시나리오#

소액 결제 앱에서 1,000건의 전송을 L2에서 즉시 확정 처리 후 1개의 배치로 L1에 게시 사용자 경험은 빠른 확정성 제공, 서비스는 L1 수수료를 배치 단위로 나눠 단건당 비용을 크게 절감 규모가 커질수록 단건 평균 비용 하락, 다만 피크 타임에는 게시 지연 가능성 존재

베스트 프랙티스#

결제·거래 핵심 경로는 감사된 라이브러리 활용 브릿지·출금 경로는 가시적 SLA와 대체 루트 제공 중요 상태 변경은 L1 루트 상태 기준으로 재동기화 가능하게 설계 테스트넷에서 L1 가스 스파이크와 시퀀서 장애를 가정한 혼합 부하 테스트 수행

참고 구분 팁#

솔라나는 고성능 L1로 롤업이 아님 폴리곤은 여러 제품군이 존재하며 PoS 체인은 사이드체인에 가깝고 zkEVM·Miden 등은 L2 범주 분류 시 합의 보안 상속 주체와 DA 위치를 먼저 확인 권장
