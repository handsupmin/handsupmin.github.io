Prisma OrderByWithAggregationInput로 집계 기반 정렬하기와 올바른 사용 맥락

November 6, 2025

개요#

OrderByWithAggregationInput은 일반 필드뿐 아니라 _count, _avg, _sum 같은 집계 결과를 기준으로 정렬을 정의하는 Prisma 입력 타입임 핵심은 이 타입이 findMany가 아니라 groupBy에서 사용하는 것이라는 점임 집계 결과를 섞어 정렬하려면 groupBy 컨텍스트에서만 가능하며, 일반 목록 조회(findMany)에서는 관계 수 기준(_count) 정도만 지원하고 이는 별도 타입 경로를 통해 처리됨

핵심 개념#

  • 대상 연산
    • groupBy에서만 OrderByWithAggregationInput 사용 가능
    • 일반 목록 조회는 ModelOrderByWithRelationInput 또는 OrderByRelationAggregateInput 사용
  • 정렬 기준 구성
    • 스칼라 필드 정렬 + 집계 결과 정렬을 함께 배열로 전달 가능
    • orderBy는 하나 또는 배열, 우선순위는 배열 순서대로 적용
  • 타입 안전성
    • 집계 키(_count, _avg, _sum, _min, _max)와 필드명이 스키마와 정합되도록 컴파일 타임 검증됨

사용법과 예시#

아래 예시는 Inventory 모델을 기준으로 설명함

  1. 타입 정의와 동적 정렬 입력 준비
import { Prisma } from '@prisma/client'

type OrderQuery = Prisma.InventoryOrderByWithAggregationInput
  1. groupBy에서 필드 + 집계 기준 복합 정렬
const where: Prisma.InventoryWhereInput = { status: 'active' }

const orderBy: OrderQuery[] = [
  { itemId: 'asc' },            // 일반 필드 정렬
  { _count: { _all: 'desc' } }, // 전체 카운트 기준 정렬
]

const rows = await prisma.inventory.groupBy({
  by: ['itemId', 'status'],
  where,
  orderBy,
  take: 10,   // groupBy에서 take/skip 사용 시 orderBy 필수
  skip: 0,
})
  • groupBy는 by에 명시한 필드를 기준으로 결과를 묶음
  • orderBy에 스칼라 필드와 집계를 함께 나열 가능
  • _count는 특정 필드 또는 _all 지정 가능, _avg와 _sum은 숫자 타입 필드만 대상
  1. 평균/합계 같은 수치 집계를 기준으로 정렬
const orderByAvgSum: OrderQuery[] = [
  { _avg: { price: 'asc' } },
  { _sum: { quantity: 'desc' } },
]

findMany와의 차이 및 대안#

  • findMany에서는 OrderByWithAggregationInput 사용 불가
  • 관계 개수로 정렬이 필요하면 OrderByRelationAggregateInput 경로 사용
// 예: 사용자 목록을 게시물 수 기준 내림차순
const users = await prisma.user.findMany({
  orderBy: { posts: { _count: 'desc' } },
})
  • 스칼라 집계(_avg, _sum 등)를 기준으로 레코드 자체를 정렬해야 한다면 findMany가 아니라 groupBy로 전환 필요

주의사항#

  • groupBy의 orderBy에 지정하는 스칼라 필드는 by에 포함되어야 함
  • _avg, _sum, _min, _max 대상 필드는 해당 연산에 적합한 타입이어야 함
  • groupBy에서 take/skip 사용 시 반드시 orderBy를 함께 지정해야 함
  • 집계 기준 정렬은 결과 형태가 groupBy 출력 스키마로 바뀜에 유의, 기존 엔드포인트의 응답 스키마가 달라질 수 있음

정리#

  • OrderByWithAggregationInput은 집계 결과 기반 정렬을 위한 groupBy 전용 타입임
  • 스칼라 필드와 집계를 배열로 조합해 우선순위 정렬 구성 가능
  • findMany에서는 관계 개수 정렬만 지원하며 스칼라 집계 정렬은 불가, 필요 시 groupBy로 설계 전환 권장
  • 타입 안전성을 유지하면서 동적 정렬을 구성하려면 입력 타입을 명시적으로 선언하고 배열 기반으로 조합 관리 권장
