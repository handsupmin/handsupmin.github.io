영지식 증명 입문 1편: NP·IP·AM, 알리바바 동굴, 3색 칠하기, Fiat–Shamir

November 5, 2025

개요#

블라인드 서명은 내용을 숨긴 채 서명을 얻는 특성 덕에 영지식성의 일부를 직관적으로 보여줌. 본 글은 영지식 증명 전반을 개념 정리 관점에서 다룸. NP 증명 시스템과 대화형 증명 시스템, AM 프로토콜과 IP=PSPACE 결과, 직관적 비유인 알리바바의 동굴, 대표적 구현 예시 몇 가지, 그리고 비대화형 영지식 증명과 Fiat–Shamir 휴리스틱까지의 흐름 정리

NP 증명 시스템#

결정 문제는 예 또는 아니오로 답할 수 있는 문제를 의미함. 명제 x를 이진 문자열로 보는 형식 언어 L ⊆ {0,1}*의 원소로 놓고, 증명자 P는 x ∈ L임을 보이며 검증자 V는 이를 다항 시간에 확인하는 역할을 가짐

  • P와 V를 결정론적 튜링 기계로 모델링
  • P는 무한 계산 가능, V는 다항 시간 검증 가능
  • P는 명제 x에 대한 증거 w를 생성해 V에게 제공, V는 다항 시간 내 예 또는 아니오 판정

NP 문제는 비결정론적 튜링 기계로 다항 시간에 풀 수 있는 판정 문제 집합이며, 결정론적 튜링 기계로는 다항 시간 검증 가능

대화형 증명 시스템과 조건#

대화형 증명은 P와 V가 상호작용하며 판정하는 틀. V는 확률적 튜링 기계, P는 계산 무제한으로 모델링하는 경우가 일반적

단방향 상호작용의 개념적 흐름

  • P와 V는 명제 x를 공유
  • P가 증거 π를 전송
  • V가 π를 근거로 x ∈ L 여부 판정

충족해야 할 두 속성

  • 완전성 x ∈ L이면 정직한 P의 증명을 정직한 V가 받아들일 확률이 높음. 충분히 큰 n와 모든 k에 대해 최소 1 − 1/n^k 이상으로 설정하는 것이 표준적 관례
  • 건전성 x ∉ L이면 어떤 P의 속임수에도 정직한 V가 받아들일 확률이 낮음. 충분히 큰 n와 모든 k에 대해 최대 1/n^k 이하로 설정하는 것이 표준적 관례

완전성과 건전성은 오류 확률을 임의로 작게 줄일 수 있도록 반복 구성 가능함

Arthur–Merlin 프로토콜#

아서–멀린 프로토콜은 검증자의 무작위성 사용을 전제로 한 대화형 증명의 표준화된 변형. 증명자 멀린 M은 계산 무제한, 검증자 아서 A는 공정한 난수에 접근 가능하다고 가정

  • 완전성 x ∈ L이면 정직한 상호작용에서 A가 받아들일 확률 ≥ 2/3
  • 건전성 x ∉ L이면 어떤 속임수에도 A가 받아들일 확률 ≤ 1/3

위 확률 경계는 반복으로 임의의 수준까지 증폭 가능. 아서의 무작위 선택이 판정의 핵심이며 멀린의 주장을 오로지 문제 구조와 무작위성으로 검증한다는 점이 특징

IP와 IP = PSPACE#

검증자가 확률적, 증명자가 계산 무제한이며 다항 횟수 상호작용이 허용될 때 풀 수 있는 판정 문제 집합을 IP라 정의

중요 사실 IP = PSPACE로 알려져 있음. 결정론적 혹은 비결정론적 튜링 기계가 다항 공간 내에서 풀 수 있는 문제 집합과 동일함. 표준적으로 알려진 포함 관계는 다음과 같음

  • P ⊆ NP ⊆ PSPACE

모든 포함이 엄격한지 여부는 미해결이며 특히 P 대 NP 문제는 여전히 열린 문제

Zero Knowledge의 직관과 조건#

대화형 증명은 보통 참을 설득하기 위해 많은 정보를 내보냄. 영지식 증명은 명제가 참이라는 사실 외에 추가 정보를 노출하지 않음. 즉 검증자는 참 여부만 알게 되고 그 외 지식은 얻지 못함

영지식 증명의 조건

  • 완전성 명제가 참이면 정직한 P가 정직한 V를 설득 가능
  • 건전성 명제가 거짓이면 어떤 속임수에도 V가 받아들이지 않음
  • 영지식성 명제가 참일 때 V는 참 여부 외에는 아무 정보도 습득하지 않음. 형식적으로는 시뮬레이터가 생성한 가상 대화와 실제 대화가 구분되지 않아야 함

알리바바의 동굴 비유#

동굴의 A, B 출입구와 중앙의 비밀문이 있다고 가정. 비밀번호를 알면 A로 들어가 B로 나오거나 그 반대가 가능. 모르면 진입한 출구로 다시 나오는 것만 가능

프로토콜 흐름

  • 증명자 P는 A 또는 B 중 무작위로 진입
  • 검증자 V는 임의의 출구로 나오라고 요청
  • P는 요청된 출구로 나옴

P가 비밀을 모르면 요구를 맞출 확률은 1/2. 여러 라운드를 반복하면 속임수로 전 라운드를 통과할 확률은 지수적으로 감소. 동시에 V는 실제 비밀번호 자체를 배우지 않음. 따라서 완전성, 건전성, 영지식성의 직관을 모두 만족

구현 예시 1: 그래프 3색 칠하기의 영지식 증명#

문제 정의 요약. 인접한 정점이 서로 다른 색이 되도록 3가지 색으로 그래프를 칠하는 과제. 3색 칠하기는 NP-완전으로 알려짐. P는 유효한 3색 칠하기를 알고 있으나 색칠법 자체를 드러내지 않고 싶다는 요구 조건

기본 아이디어

  • P가 각 정점의 색을 커밋먼트로 고정. 커밋은 개봉 전까지 내용 은닉, 개봉 후에는 변경 불가 특성을 가져야 함. 해시 커밋먼트나 페더슨 커밋먼트 등 표준 기법 가정
  • V가 임의의 간선을 질의
  • P가 해당 간선의 양 끝 정점에 대해 커밋을 개봉하여 두 색이 서로 다름을 증명
  • 위 과정을 다수의 간선에 반복하여 신뢰도 확보

영지식성 주의점

  • 매 라운드 색 이름을 무작위 치환하여 공개해야 함. 그렇지 않으면 반복 개봉으로 전체 색칠이 누적적으로 노출될 수 있음
  • 적절한 은닉성의 커밋을 사용하고 색 치환을 매 라운드 적용하면 검증자가 다항 시간에 유의미한 색칠법을 복구하는 알고리즘이 존재하지 않는 것으로 알려짐. 이 수준을 계산적 영지식성이라 부름

귀결. 3색 칠하기가 NP-완전이므로 임의의 NP 문제는 다항 시간 환원을 통해 이 구조로 영지식 대화형 증명 구축 가능

구현 예시 2: 이산대수 기반 대화형 증명#

표준적인 이산대수 증명 스케치. 소수 p와 생성원 a가 주어진 곱셈군 Z_p^*에서 비밀 x에 대해 공개 b ≡ a^x mod p를 제공하는 상황. 목표는 x를 드러내지 않고 지식 보유를 증명하는 것

프로토콜 흐름

  • P가 무작위 r를 택해 R ≡ a^r mod p를 전송
  • V가 이진 도전 e ∈ {0,1}를 전송
  • P가 e = 0이면 t = r, e = 1이면 t = r + x mod (p − 1)을 전송
  • V가 e = 0이면 a^t ≡ R mod p, e = 1이면 a^t ≡ b·R mod p를 검증
  • 여러 라운드 반복으로 오류 확률 축소

위 구조는 지식 추출 가능성과 완전성·건전성·영지식성의 직관을 충족. 도전 공간을 확장하고 해시로 치환하면 표준 식별 프로토콜로 귀결됨

구현 예시 3: Fiat–Shamir 식별 프로토콜#

합성수 모듈러스 n = p·q에 대해 비밀 s_i와 공개 v_i ≡ s_i^2 mod n를 사용하는 식별 스킴. P만 s_i를 알고 V는 v_i와 n을 알고 있다고 가정

프로토콜 흐름의 한 형태

  • P가 무작위 r와 부호 s ∈ {−1, 1}을 택해 x ≡ s · r^2 mod n을 전송
  • V가 도전 비트열 a_1, …, a_k를 전송
  • P가 y ≡ r · s_1^{a_1} · s_2^{a_2} · … · s_k^{a_k} mod n을 전송
  • V가 y^2 ≡ ± x · v_1^{a_1} · v_2^{a_2} · … · v_k^{a_k} mod n을 확인. 부호는 s 처리에 따른 ±를 의미

k 라운드 반복 또는 일괄 도전으로 건전성을 확보. 이 구조는 이후 비대화형 전환의 기반이 됨

비대화형 영지식 증명 NIZK#

대화형 영지식 증명은 반복 통신이 필요해 시스템 통합과 네트워크 환경에서 비효율이 발생. 검증자와의 실시간 왕복 없이 증명 산출물만으로 검증 가능한 비대화형 영지식 증명이 필요

핵심 과제는 도전값을 누가 정하느냐의 문제. 대화형에서는 검증자가 무작위 도전을 제공해 P의 조작 여지를 차단하지만, 비대화형에서는 이를 대체하는 메커니즘이 필요

Fiat–Shamir 휴리스틱과 랜덤 오라클#

Fiat–Shamir 휴리스틱은 대화형 프로토콜의 검증자 도전을 해시 함수로 대체하는 일반적 기법. 직관은 다음과 같음

  • 도전값 e를 검증자의 무작위 대신 e = H(공개 파라미터, 명세, 스테이트, 커밋 R, 메시지 등)으로 정의
  • 누구나 동일 입력으로 동일 해시 출력을 재현 가능해야 하며, 해시는 랜덤 오라클처럼 동작한다고 가정

랜덤 오라클 모델에서는 해시를 완전한 무작위 함수로 이상화. 이 가정하에 많은 대화형 영지식 프로토콜이 비대화형으로 안전하게 전환 가능. 실무에서는 충돌내성·예측불가능성이 강한 해시를 선택하고, 전환 규칙과 도전 입력 구성 요소를 엄격히 고정해 재현성과 변조 불가성을 보장함

한계. 랜덤 오라클 모델의 보안 증명이 표준 모델 보안을 자동으로 보장하지는 않음. 또한 구현 시 도전 입력에 포함되는 모든 컨텍스트를 명시적으로 해싱하지 않으면 재사용 공격이나 문맥 오염 위험 존재

정리#

영지식 증명은 완전성, 건전성, 영지식성을 동시에 달성하는 대화형 증명 패턴에서 출발. NP 프레임과 AM·IP 결과는 이론적 경계를 제공하며, 알리바바 동굴과 3색 칠하기, 이산대수, Fiat–Shamir 등은 직관과 구현의 교량 역할을 함. 비대화형 전환에서는 해시를 통한 도전 생성으로 실용성을 확보하되 모델 가정과 입력 구성의 엄격함이 중요
