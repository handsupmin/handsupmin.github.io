Controller와 Service에서 Validation을 나누는 기준

December 8, 2025

개요#

입력 유효성 검사는 레이어 별 책임을 나눌 때 유지보수성과 테스트 용이성이 올라감 컨트롤러는 요청 스키마와 데이터 형태를 보증, 서비스는 도메인 규칙과 상태 변화를 보증하는 역할

컨트롤러에서 하는 검증#

  • 타입 검사 @IsString, @IsNumber
  • 포맷 검사 @IsEmail, @IsJWT, @IsEthereumAddress
  • 길이·범위 @MaxLength, @Min, @Max
  • 필수/선택 @IsOptional
  • Enum 값 @IsEnum
  • 기본 sanitization 및 단순 normalize

서비스에서 하는 검증#

  • 비즈니스 규칙 충족 여부
  • 데이터 존재 여부 확인
  • 권한과 소유권 검증
  • 현재 상태에서 변경 가능 여부
  • 여러 조건을 조합한 복합 검증

위처럼 분리하면 관심사 분리와 단일 책임 원칙을 충족하는 기본 기준 충족
