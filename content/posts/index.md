> TL;DR
- 서버, API, 데이터베이스 등 백엔드 핵심 용어 정리
- 캐시와 미들웨어 같은 성능 및 구조 관련 개념 포함
- JWT, ORM, 컨테이너 등 최신 개발 환경에서 자주 쓰이는 용어 설명서버: 클라이언트 요청을 처리하고 데이터를 제공하는 컴퓨터 또는 프로그램. API(Application Programming Interface): 서로 다른 소프트웨어 간에 기능이나 데이터를 주고받기 위한 인터페이스. 데이터베이스: 데이터를 체계적으로 저장하고 관리하는 시스템. 캐시: 자주 사용하는 데이터를 임시로 저장해 빠르게 접근할 수 있게 하는 메모리 공간. REST: HTTP 프로토콜을 기반으로 한 대표적인 API 설계 방식. JWT(Json Web Token): 사용자 인증과 권한 부여에 쓰이는 토큰 기반 인증 방식. ORM(Object-Relational Mapping): 객체 지향 프로그래밍 언어에서 관계형 데이터베이스를 쉽게 다루기 위한 기술. 미들웨어: 요청과 응답 사이에서 특정 기능을 수행하는 소프트웨어 계층. 로드 밸런서: 여러 서버에 부하를 분산시켜 서비스 안정성을 높이는 장치. 컨테이너: 애플리케이션과 그 실행 환경을 패키징해 어디서나 동일하게 실행할 수 있게 하는 기술.
